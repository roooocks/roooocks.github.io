## Python

### pyenv 설치
[pyenv](https://github.com/pyenv/pyenv)를 사용한다.
- 홈에서 클론 따기 : `git clone https://github.com/pyenv/pyenv.git ~/.pyenv`
- shell 환경에 pyenv 실행을 설정한다. 무슨 shell이냐에 따라 방법이 달라지는데, 그건 git 페이지를 참고하자.

```bash
# .bashrc file
...
echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.bashrc
echo '[[ -d $PYENV_ROOT/bin ]] && export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(pyenv init - bash)"' >> ~/.bashrc
...
```

- shell 재시작 : `source ~/.bashrc`
- 버전 확인하기 : `pyenv --version`
<br>

### 버전 설치
```bash
# 아래 2가지를 미리 진행해야 한다.
# 만약 Python 3.14 이상을 빌드한다면 libzstd-dev 설치가 필수다.
sudo apt update

sudo apt install make build-essential libssl-dev zlib1g-dev \
libbz2-dev libreadline-dev libsqlite3-dev curl git \
libncursesw5-dev xz-utils tk-dev libxml2-dev libxmlsec1-dev libffi-dev liblzma-dev

pyenv versions
pyenv install --list
pyenv install <version>
pyenv uninstall <version>
```
<br>

### 사용 방법
1. 프로젝트 혹은 공부할 위치로 이동
2. 아래 방식으로 설정

```bash
pyenv global <version>

# pyenv-version 파일 사용. 위치 진입 시 자동 설정(쉘에 pyenv init 설정 필요)
pyenv local <version>

# 현재 쉘에만 임시 적용
pyenv shell <version>
```

## Node.js

### nvm 설치
[nvm](https://github.com/nvm-sh/nvm)을 사용한다.
```bash
# 없으면 설치
sudo apt curl
sudo apt wget

# 설치 혹은 업데이트 가능
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.4/install.sh | bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.4/install.sh | bash

# 버전 나오면 성공
nvm -v

# nvm은 실행 파일로 동작하는게 아닌 쉘 함수 형태로 동작한다.
# 때문에 현재 쉘에 활성화를 시켜주는 코드를 삽입해야 nvm을 명령어처럼 사용할 수 있다.
# .bashrc에 넣어주면 된다.
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"  # This loads nvm
[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"  # This loads nvm bash_completion

# XDG 표준에 맞출거면 아래거 사용(이게 최근거긴 하다.)
# 이러면 무조건 ~/.nvm 파일 사용을 안해도 된다.
export NVM_DIR="$([ -z "${XDG_CONFIG_HOME-}" ] && printf %s "${HOME}/.nvm" || printf %s "${XDG_CONFIG_HOME}/nvm")"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh" # This loads nvm
```
<br>

### 버전 설치
```bash
# 보통 node라는 alias를 쓰면 최신걸로 받는다.
nvm install <version> or <alias>
node -v
```
<br>

### 사용 방법
```bash
# global, local이 따로 없다.
# default 설정하면 그게 global이다.
nvm alias default v<version>

# .nvmrc 파일에 버전/lts만 넣으면 해당 버전을 쉘에서 사용할 수 있게 해준다.
nvm use

# 버전 선택
nvm use <version> or <alias>
```

## Ruby

### rbenv 설치
[rbenv](https://github.com/rbenv/rbenv)를 사용한다. <br>
rvm도 있긴한데, 블로그 쓰면서 알게되가지고 나중에 쓸 일 있으면 작성하겠다. (rbenv보다 무겁다고 한다.) <br>
```bash
# 아래 3가지 진행
sudo apt install rbenv
rbenv init
rbenv -v

# 만약 설치가 안된다면 사용
sudo apt update
sudo apt install build-essential libssl-dev libyaml-dev libreadline-dev zlib1g-dev libncurses-dev libffi-dev autoconf bison
```
<br>

### 버전 설치
```bash
# 모든 루비 버전 확인
rbenv versions

# 현재 루비 버전 확인
rbenv version

# list latest stable versions:
rbenv install -l

# list all local versions:
rbenv install -L

# install a Ruby version:
rbenv install 3.1.2

# 설치 후 재시작
rbenv rehash
```
<br>

### 사용 방법
```bash
rbenv global <version>
rbenv local <version>	# 사용할 프로젝트 폴더에 사용

# bundler는 따로 그리고 ruby가 활성화 된 곳에 설치해야 한다.
gem install bundler
gem env home
bundler # 현재 설치된 ruby 패키지 확인
bundler intsall <name>
```
