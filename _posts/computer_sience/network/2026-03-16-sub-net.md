## 무엇인가?
기존의 Class 기반 IP 주소 체계는 <ins>네트워크 크기를 유연하게 나누기 어렵다는 문제</ins>가 있다. <br>
예를 들어 Class C는 한 네트워크에 최대 254개의 Host를 사용할 수 있지만, 실제 환경에서는 이보다 훨씬 작은 규모로 나누어야 하는 경우가 많다. <br>
이 문제를 해결하기 위해 등장한 개념이 바로 **subnet(서브넷)**과 **subnet mask(서브넷 마스크)**이다.
<br><br>



## subnet
하나의 네트워크를 더 작은 네트워크 단위로 나눈 것을 **subnet**이라고 한다. <br>
즉, 기존에는 하나의 네트워크로 사용하던 주소 범위를 여러 개의 작은 네트워크로 분할해서 사용하는 개념이다.
<br><br>



### 왜 사용하는가
예를 들어 `210.102.223.0` 네트워크가 있다고 가정해보자. <br>
이 네트워크를 4개의 공간(예시: 교실)으로 나누고 싶다면, 하나의 네트워크를 그대로 사용하는 것이 아니라 <ins>여러 개의 subnet으로 나누어 사용</ins>할 것이다. <br>
이때 중요한 점은 **host의 영역 일부를 network 영역으로 확장해서 나눈다는 것**이다.

```
원래: network + host
변경: network + subnet + host
```
{: file='이렇게 구조가 확장' }

이를 통해 얻을 수 있는 이점은 아래와 같다.
- 네트워크를 효율적으로 분할 가능
- 관리 단위 분리 가능
- 불필요한 트래픽 감소
<br><br>



## subnet mask
<ins>IP 주소에서 어디까지가 network이고 어디부터 host인지 구분하는 기준</ins>이다. <br>
예를 들어서 `192.168.0.1/24`와 `192.168.0.1/26`에 있는 `/n`은 <ins>앞에서부터 n비트가 network 영역이라는 의미</ins>이다.
- `/24` → 앞에서 24비트까지가 network
- `/26` → 앞에서 26비트까지가 network (더 잘게 쪼갬)

#### /24 설명
IP가 `192.168.0.10/24`고 subnet mask가 `255.255.255.0(11111111.11111111.11111111.00000000)`일 경우 <br>
- 앞에서 24비트까지가 network
- 나머지 8비트가 host

각각의 주소와 범위는 아래와 같아진다.
- network 주소 : `192.168.0.0`
- host 범위 : `192.168.0.1 ~ 192.168.0.254`
- broadcast 주소 : `192.168.0.255`
<br>

#### /26 설명 (서브넷 분할)
IP가 `192.168.0.10/26`고 subnet mask가 `255.255.255.192(11111111.11111111.11111111.11000000)`일 경우 <br>
- 앞에서 26비트까지가 network
- 나머지 6비트가 host

만약 기존에 /24 네트워크(256개의 주소)를 사용하고 있었다는 가정 하에 이를 4개의 /26 서브넷으로 나눌 수 있다. subnet 목록은 아래와 같다. <br>
1. `192.168.0.0/26`
 - host: `192.168.0.1 ~ 192.168.0.62`
 - broadcast: `192.168.0.63`
2. `192.168.0.64/26`
 - host: `192.168.0.65 ~ 192.168.0.126`
 - broadcast: `192.168.0.127`
3. `192.168.0.128/26`
 - host: `192.168.0.129 ~ 192.168.0.190`
 - broadcast: `192.168.0.191`
4. `192.168.0.192/26`
 - host: `192.168.0.193 ~ 192.168.0.254`
 - broadcast: `192.168.0.255`
<br><br>



## 핵심 정리
- subnet은 **하나의 네트워크를 나눈 작은 네트워크**이다.
- subnetting은 **host 비트를 network 비트로 확장하여 네트워크를 분할하는 과정**이다.
- subnet mask(/n)는 **network와 host를 구분하는 기준**이다.
- 라우팅은 **host가 아닌 network 기준으로 이루어진다.**

### CIDR
예제를 보면 /26과 `255.255.255.192(11111111.11111111.11111111.11000000)`가 같이 나온다. <br>
좀 찾아보니 /26같은 표현은 CIDR이라고 subnet mask의 간단한 표현이라고 한다. 옆에 엄청 길다란 표현이 옛날 표현인 듯 하다.
