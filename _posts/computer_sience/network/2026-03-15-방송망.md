## 방송망
- 여러 노드가 하나의 통신 매체(공기, 수중, 선로, 광케이블 등)를 공유하여 데이터를 전송하는 네트워크 구조
- 공유 시 발생할 수 있는 **충돌(collision)**을 효과적으로 회피하거나 방지해야 한다.
- 여러 노드가 매체를 공유하기 위해 다음과 같은 다중접속 방식을 사용한다.
  - 시간(time/시분할) = 시간을 정해놓고 사용
  - 주파수(frequency) = 같은 시간에 데이터 동시 사용 가능(주파수가 서로 다름/주파수 분할 방식)
  - 코드 = 같은 시간, 같은 주파수지만 코드가 달라 동시 사용 가능
<br><br>



## MAC (Media Access Control)
MAC은 <ins>여러 노드가 하나의 통신 매체를 공유할 때
매체 사용 순서를 제어하고 충돌을 방지하거나 처리</ins>하기 위한 **프로토콜**이다. <br>
이 게시물에서는 학교에서 설명한 방식만 작성한다. (최신의 방식이 있다면 새 게시물에다 그때마다 작성하자)

### ALOHA
- 교수님 말로는 하와이 대학에서 사용했다고...
- 핵심은 <ins>전송하고 싶을 때 바로 전송하고 충돌 발생 시 일정 시간 후 다시 전송</ins>이다.
- 학교에서 배운 방식은 2가지 주파수의 사용 방식이다.
  - f1 : 메인 → 터미널
  - f2 : 터미널 → 메인
  - 동작 방식
    - 각 터미널이 정보를 전송한 후 main의 응답 대기
    - 메인 응답 시 Yes, collision 발생(복수 터미널이 동시 전송) 시 재전송
- 처리율이 대략 18%라고 한다. (5번 중 1번 성공)
<br>

### slot ALOHA
- 정보 전송을 정해진 시점(time slot)에서만 수행 (시간관리, 동기화)
  - 시간관리는 망 자체가 관리한다고 한다.
- 처리율이 대략 37%로 향상되었다고 한다. (대신 복잡해짐)

### CSMA (Carrier Sense Multiple Access)
- 전송 매체를 **먼저 검사**한 후
  - 사용 중일 시 대기 후 다시 검사
  - 사용 안할 시 전송 개시
- CSMA는 ALOHA 방식에서 **먼저 검사** 과정이 추가된 방식이라 ALOHA보다 효율이 좋다.
<br>

### CSMA/CD (CSMA + Collision Detection)
- 전송 중에 충돌 발생을 **계속 점검(Collision Detection)**하며, 충돌 감지 시 전송을 중단하고 일정 시간 후 다시 전송한다.
- 교수님 말로는 통신 장비를 보면 collision 불이 계속 점멸하는 것을 볼 수 있는데
계속 충돌을 감지하기 때문이라고 한다.
(AI 말로는 과거 Ethernet Hub 장비에서 흔히 볼 수 있었다고 한다.)
