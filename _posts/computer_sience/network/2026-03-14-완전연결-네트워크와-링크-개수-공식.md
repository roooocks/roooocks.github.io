## 완전연결 네트워크 개념
네트워크에서 host 간의 통신 구조를 표현할 때 보통 **선(연결선, 링크)**으로 연결 관계를 나타낸다. <br>
이때 모든 host(노드)가 서로 직접 연결되어 있는 구조를 **완전연결 네트워크(Complete Connection)**라고 한다. 즉, 어떤 노드든 다른 모든 노드와 직접 통신할 수 있도록 각각 연결된 상태를 의미한다.

예를 들어 host가 4개라면 다음과 같은 형태가 된다.
- A ↔ B
- A ↔ C
- A ↔ D
- B ↔ C
- B ↔ D
- C ↔ D

모든 노드가 서로 연결되기 때문에 연결선의 수가 빠르게 증가하는 특징이 있다.
<br><br>



## 필요한 연결선 개수
완전연결 네트워크에서 필요한 연결선(링크)의 개수는 다음 식으로 계산한다.
```
n(n−1) / 2
```
{: file='계산식' }

n = host(node)의 개수이고, 식은 분수 형태로 표현하는 것이 일반적이라고 한다.
<br><br>



## 예시
Host 개수 (n)	필요한 연결선

| Host 개수 (n) | 필요한 연결선 |
|---|---|
| 2 | 1 |
| 3 | 3 |
| 4 | 6 |
| 5 | 10 |

예를 들어 host가 4개라면 6개의 연결선이 필요
```
4(4-1) / 2 = 6
```
<br><br>



## 특징
모든 노드가 직접 연결되기 때문에 통신 지연이 거의 없다. 하지만 노드가 증가할수록 연결선이 기하급수적으로 증가한다. <br>
따라서 <ins>실제 대규모 네트워크에서는 비용과 구조 문제</ins> 때문에 잘 사용되지 않는다.



## 정리
완전연결 네트워크는 모든 노드가 서로 직접 연결된 구조를 의미한다. <br>
이때 필요한 연결선의 수는 n(n-1)/2 공식으로 계산할 수 있으며, 노드 수가 증가할수록 연결선의 수가 급격히 증가하는 특징이 있다.
