## 간단 정리
우선은 **간단한 개념만 정리한다.**
나중에 실제로 사용하거나 활용할 일이 생기면 따로 게시물을 올리던가 해서 더 자세하게 정리할 예정이다.

## ISO (International Organization for Standardization)
학교에서는 International Standard Organization이라고 하는데, 뭐가 맞는지는 모르겠다. <br>
어쨋든 ISO는 **국제 표준 기구**로 다양한 분야의 표준을 만든다. 대표적인 분야는 다음과 같다.
- 경영
- 토목 / 건축
- IT (통신)
- 등등

이 중 **IT 통신 분야에서 매우 유명한 계층 구조 모델**이 존재하는데 그것이 **OSI 7계층**이다.
<br><br>



## OSI (Open Systems Interconnection)
OSI는 네트워크 통신을 **7개의 계층(Layer)**으로 나눈 모델이다. (그래서 OSI model이라고도 한다.) 보통 **OSI 7 Layer** 또는 **OSI 7계층**이라고 부른다.

### 상위 계층
이 세 계층은 **Application 계층**이나 **상위 계층**으로 묶어서 보기도 한다.

| Layer | 이름 |
|---|---|
| L7 | Application (응용) |
| L6 | Presentation (표현) |
| L5 | Session |

<br>

### 중간 계층
이 계층의 Network(L3)부터 <ins>다른 네트워크 간 통신을 담당</ins>한다. (IP, Routing 등)

| Layer | 이름 | 역할 |
|---|---|---|
| L4 | Transport (전송) | 흐름 제어, 에러 제어 |
| L3 | Network | 주소 관리, 경로 설정 / 유지 / 해제, Routing |

<br>

### 하위 계층
이 두 계층만 있어도 **1:1 통신은 가능**하다. 좀 더 정확히는 <ins>같은 네트워크 내부에서는 1:1 통신이 가능</ins>하다.

| Layer | 이름 | 역할 |
|---|---|---|
| L2 | Data Link | 흐름 제어, 에러 제어 |
| L1 | Physical | 물리적 신호 전달 |

<br>

### Physical Layer 특징
Physical Layer는 다음과 같은 **물리적 특성**을 정의한다. <br>
**기계적 특성**
- 모양
- 치수
- 재질
- 색상
- 개수

**전기적 특성**
- 전압
- 전류
- 주파수
- 극성

전송 매체로는 <ins>케이블, 광섬유, 무선</ins> 등이 있다.
<br><br>



## OSI 7계층에서 중요한 부분

### 1. L1과 L2의 관계
L1에서 **신호를 전송**하면, L2에서 **그 신호의 구조를 정의**한다. 데이터 전송 구조는 다음과 같다.
```
도착 ← 주소 / 길이 / 내용 / CRC ← 출발
```
{: file='데이터 전송 구조' }

여기서 **CRC는 에러 검출 코드**이며 반드시 **프레임(데이터)의 마지막에 붙어서 전송**된다.
<br>

### 2. L2와 L4의 흐름제어 / 에러제어 차이
같은 용어를 사용하지만 **의미가 다르다.**

| Layer | 흐름제어 | 에러제어 |
|---|---|---|
| L2 | 신호 흐름 제어 | 에러 검출 |
| L4 | 네트워크(망) 흐름 제어 | 에러 복구 |

즉, L2는 **물리적 신호 수준**, L4는**네트워크 통신 수준**의 제어라고 볼 수 있다. <br>
대학교 정리라 그런데 흐름제어 부분을 자세히 설명하면 아래처럼 된다.
- L2 : 인접 노드 간 흐름 제어
- L4 : 종단 간(end-to-end) 흐름 제어
