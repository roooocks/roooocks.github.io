## IP Address
네트워크 계층(L3)에서 사용하는 프로토콜로 주소 지정과 패킷 전달을 담당한다. 라우터는 이 주소 정보를 기반으로 경로를 결정한다. <br>
교수님 말씀으로는 <ins>한정된 자원을 전 세계에서 써야</ins>하기 때문에 네트워크 주소(이하 net-id)와 호스트 주소(이하 host-id)로 구분하는 **Class 방식**을 사용한다 하시는데, AI는 초기 인터넷에서는 <ins>IP 주소를 단순하게 관리</ins>하기 위해 Class 기반 주소 체계를 사용한다고... <br>
설명에서 사용하는 IP는 IPv4로 구조는 x.x.x.x로 되어 있으며, 각 자릿수마다 0~255(2^8, 8bit)의 수를 가진다. (총 4Byte, 32bit)

### Class A
- 범위는 0.0.0.0 ~ 127.255.255.255
  - 실제 사용은 1.0.0.0 ~ 126.255.255.255라고 한다.
  - 0.~는 현재 네트워크를 의미하며, 127.255.255.255는 Loopback 주소라 일반적으로 사용을 안한다고한다.
- 2진수로 표시할 경우 <ins>맨 앞자리는 0 고정</ins>이다.
<br>

### Class B
- 범위는 128.0.0.0 ~ 191.255.255.255
- 2진수로 표시할 경우 <ins>맨 앞자리는 1 0 고정</ins>이다.
<br>

### Class C
- 범위는 192.0.0.0 ~ 223.255.255.255
- 2진수로 표시할 경우 <ins>맨 앞자리는 1 1 0 고정</ins>이다.
<br>

### Class D
- 범위는 224.0.0.0 ~ 239.255.255.255
- 2진수로 표시할 경우 <ins>맨 앞자리는 1 1 1 0 고정</ins>이다.
- multicast 용도로 사용된다고 한다.
<br>

### Class E
- 범위는 240.0.0.0 ~ 255.255.255.255
- 2진수로 표시할 경우 <ins>맨 앞자리는 1 1 1 1 0 고정</ins>이다.
- 연구 및 실험용으로 예약된 주소라고 한다.
<br><br>



## net-id, host-id의 구조
예를 들어 IP 210.102.223.0를 가진 네트워크(net-id)가 하나 있다고 하자. 그 네트워크(망) 안에는 여러 기기들이 접속해 있을 것이다. 이때 연결된 기기들 하나하나의 식별 번호(host-id)를 가지고 있다.

### net-id, host-id의 개수 구하기
C Class를 기준으로 설명한다.
- 고정된 네트워크 비트를 제외하고 가능한 net-id의 개수를 계산할 수 있다.
  1. 고정된 네트워크(110)를 제외하고 실제 식별에 사용 가능한 비트는 21bit로 총 2^21개의 네트워크 생성이 가능하다.
  2. 2^21 = 2^1 * 2^20(2^10 = 10^3 / 근사) = 2 * 10^6 = 대략 200만(뺀 근사 포함)
  3. net-id의 개수는 대략 200만

- host-id에는 클래스 식별 비트가 존재하지 않는다.
  1. 총 host 비트는 8비트, 2^8 = 256
  2. ~.0과 ~.255는 쓰지 않아서 256 - 2 = 254
  3. host-id의 개수는 실질적으로는 254개 사용
    - C 클래스의 **기본 구조에만 허용**된 이야기다.
    - <ins>subnet mask가 포함되면 host 범위가 달라</ins>지니 "그렇구나~" 하는 느낌으로만 이해하자
<br>

### IP의 사용 범위
C 클래스 기준으로 설명을 하자면 ~.0부터 ~.255까지가 범위로 되어있지만 실제로는 ~.1부터 ~.254까지만 사용해야 한다. <br>
~.0의 경우에는 **네트워크 자체를 의미**하기 때문이며, ~.255는 host단이 전부 1이라는 가정(아닐 경우 host 주소로 본다.) 하에 **broadcast address**이기 때문이다. <br>
  - ex) 172.16.1.255/16 = ...**.**00000001.11111111 = host
  - ex) 172.16.255.255/16 = ...**.**11111111.11111111 = broadcast
  - 뒤에 /16 은 subnet mask이다. 여기서는 설명하지 않는다.
<br>

핵심을 정리하면 다음과 같다.
 - 모든 IPv4 네트워크에서 host 부분이 모두 0이면 network address, 모두 1이면 broadcast address가 된다.
 - 실제 IP 주소는 **항상 2개를 제외한 주소만 사용**할 수 있다.
<br><br>



## dotted decimal IP
2진수를 10진수로 바꿔놓은 IP다. 우리가 흔히 보는 192.168.0.1 같은거 말이다. <br>
잠깐 도메인 이야기를 하자면, Domain name은 DNS 서버를 통해 해당하는 IP 주소를 조회할 수 있다. 이때 IP 주소는 dotted decimal 형태로 표현된다.
