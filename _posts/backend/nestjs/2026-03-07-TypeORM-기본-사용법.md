## TypeORM이란
**TypeORM**은 TS 환경에서 **데이터베이스를 쉽게 다루도록 도와주는 ORM 라이브러리**로 쉽게 말하면 **TS 코드로 DB 테이블을 만들고 CRUD 작업을 할 수 있도록 해주는 도구**다. <br>
최근에는 **Prisma**를 많이 사용하는 추세라고 한다.
<br>

### 설치
TypeORM을 NestJS에서 사용하려면 아래 패키지를 설치한다.

```bash
# 사용할_DB는 npm에서 검색해서 찾으면 된다.
npm install typeorm @nestjs/typeorm 사용할_DB

# postgresql 기준
npm install typeorm @nestjs/typeorm pg
```

여기서 역할은 다음과 같다.
- **typeorm** = 실제 ORM 기능을 담당하는 핵심 라이브러리
- **@nestjs/typeorm** = NestJS에서 TypeORM을 쉽게 사용할 수 있도록 도와주는 패키지
<br><br>



## NestJS에서 TypeORM 설정
NestJS에서는 **module.ts 파일에서 TypeORM을 설정**한다. 프로젝트 전체에서 DB를 사용해야 하므로 보통 **AppModule에 연결한다.**

### TypeOrmModule.forRoot()
`forRoot()`는 **프로젝트 전체에 TypeORM을 연결하는 설정**이다.

```ts
@Module({
  imports: [
    PostsModule,
    TypeOrmModule.forRoot({
      type: 'postgres', // db 타입
      host: '127.0.0.1',
      port: 5432,
      username: 'postgres',
      password: 'postgres',
      database: 'postgres',
      entities: [PostsModel], // DB와 연결될 Entity
      synchronize: true, // 코드와 DB 구조 자동 동기화
    }),
  ],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {}
```
{: file='forRoot의 예제이자 app.module.ts 파일 내용' }
<br>

### synchronize 옵션
```ts
synchronize: true
```

TypeORM 코드와 실제 DB 테이블 구조를 **자동으로 동기화**한다.

권장 사용 방식
- 로컬 개발 = `true`
- 프로덕션 = `false`

프로덕션에서 `true`로 설정하면 **테이블 구조가 의도치 않게 변경될 위험**이 있다.
<br><br>



## Entity (테이블 정의)
위 `app.module.ts`코드에서 `entities`라는 리스트를 확인할 수 있다. 여기에는 **DB 테이블과 연결될 Entity 클래스**를 넣는다. <br>
Entity는 아래처럼 작성한다.

```ts
import { Column, Entity, PrimaryGeneratedColumn } from 'typeorm';

@Entity()
export class PostsModel {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  author: string;

  @Column()
  title: string;

  @Column()
  content: string;

  @Column()
  likeCount: number;

  @Column()
  commentCount: number;
}
```
{: file='테이블 작성이라 생각하면 편하다.' }

- `@Entity()` = 이 클래스가 **DB 테이블이라는 의미**
- `@PrimaryGeneratedColumn()` = 자동 증가되는 **Primary Key**
- `@Column()` = 일반 컬럼

컬럼 종류의 자세한 내용은 필요하면 공식 사이트를 참고하자

이렇게 Entity를 만들고 `forRoot()`의 `entities` 배열에 넣으면 **TypeORM이 자동으로 테이블을 생성한다.**
<br>

### 폴더 구조 추천
보통 아래 구조로 작성한다.

```
posts
 ├ posts.module.ts
 ├ posts.service.ts
 ├ posts.controller.ts
 └ entity
     └ posts.entity.ts
```

## Repositroy

### Repository 개념
테이블을 정의했으면 이제 **실제로 데이터를 사용해야 한다**. 여기서 등장하는 개념이 **Repository**다. <br>
한줄로 정리하면 `특정 Entity를 다루는 전용 데이터 접근 객체`라고 할 수 있다.

역할
- DB와 애플리케이션 사이에서 **CRUD 작업 담당**
- Entity 기반으로 데이터 조회 / 생성 / 수정 / 삭제 수행

구조는 보통 다음과 같다.

```
Controller
   ↓
Service
   ↓
Repository
   ↓
Database
```
{: file='Repository 동작 구조' }
<br><br>



### Repository 사용 설정
Repository를 사용하려면 **Repository를 사용하려는 모듈에서 설정을 추가해야 한다**.
`*.module.ts` 파일에서 아래처럼 작성한다.

```ts
@Module({
  imports: [
    // Entity에 해당하는 Repository 주입
    TypeOrmModule.forFeature([PostsModel]),
  ],
  controllers: [PostsController],
  providers: [PostsService],
})
export class PostsModule {}
```
{: file='forFeature 예제' }

`forFeature()`는 **특정 Entity의 Repository를 사용할 때 필요하다**.
<br><br>



### Service에서 Repository 사용
이제 Service에서 Repository를 주입해 사용한다.

```ts
@Injectable()
export class PostsService {
  constructor(
    @InjectRepository(PostsModel)
    private readonly postsRepository: Repository<PostsModel>,
  ) {}

  async getAllPosts() {
    return this.postsRepository.find();
  }
}
```
{: file='Service에 Repository 주입 및 사용 방법' }

* `@InjectRepository(Entity)` = 해당 Entity의 Repository 주입
* `Repository<Entity>` = Entity 전용 Repository 타입
* `find()` = 모든 데이터 조회

이때 find()와 같은 Repository 함수는 대부분 **비동기 함수**이므로 `async/await`를 사용한다.
<br><br>



## 정리
NestJS에서 TypeORM 사용 흐름
1. TypeORM 패키지 설치
2. `AppModule`에서 `TypeOrmModule.forRoot()` 설정
3. Entity 작성 (테이블 정의)
4. `TypeOrmModule.forFeature()`로 Repository 등록
5. Service에서 Repository 주입 후 사용

이 구조를 이해하면 **NestJS에서 DB를 사용하는 기본 흐름을 잡을 수 있다.**
