## busboy란
Busboy는 **Node.js에서 `multipart/form-data` 요청을 스트림 기반으로 파싱하는 라이브러리**다. <br>
Node.js에서 파일 업로드를 처리할 때 보통 **multer**를 많이 사용하지만, 업로드 데이터를 **스트림 단위로 직접 제어해야 하는 경우 busboy를 사용할 수 있다**. <br>

Busboy의 핵심 특징은 다음과 같다.
- 파일을 **메모리에 모두 올리지 않는다**
- 업로드 데이터를 **스트림으로 바로 처리**
- **대용량 파일 업로드에 유리**

즉 **파일 데이터를 받는 즉시 원하는 곳으로 전달할 수 있는 구조**를 만든다.
<br><br>



## 파일 업로드 처리 흐름
```
HTTP Request (multipart/form-data)
↓
busboy
↓
file stream event
↓
pipe()
↓
filesystem / S3 / transform
```

## busboy를 사용하는 이유

### 스트림 기반 처리
busboy는 업로드 데이터를 **스트림 기반으로 처리**한다. <br>
일반적인 방식처럼 파일을 메모리에 모두 올린 뒤 처리하는 것이 아니라서 **데이터가 들어오는 즉시 처리할 수 있다**.

예를 들어 다음과 같은 처리가 가능하다.
- 업로드 => 바로 S3 전송
- 업로드 => 실시간 바이러스 검사
- 업로드 => transform 후 저장

예제 코드는 다음과 같다.

```ts
const busboy = Busboy({ headers: req.headers });

busboy.on("file", (name, file, info) => {
  const { filename, encoding, mimeType } = info;
  file.pipe(fs.createWriteStream(`./uploads/${filename}`));
});

req.pipe(busboy);
```

이 방식은 대용량 파일 처리에서도 효율적이다.
<br>

### 메모리 사용 최소화
대용량 파일 업로드에서는 메모리 사용량 관리가 중요하다.

busboy는 다음 특징을 가진다.
- 스트림 단위 처리
- 중간 버퍼 최소화
- Node.js backpressure 활용

따라서 <ins>대용량 업로드 서버</ins>에서는 **busboy 같은 low-level parser를 선호**하는 경우도 많다.
<br>

### 업로드 흐름 세밀 제어
busboy는 업로드 과정을 세밀하게 제어할 수 있다. 예를 들어 다음과 같은 로직 구현이 가능하다.
- 업로드 중 특정 조건에서 즉시 중단
- 파일 크기 제한
- 필드별 처리 로직 분기
- multipart parsing 단계 제어

즉 업로드 파이프라인에 대한 세분화된 제어(fine-grained control)가 가능하다.
<br><br>



## multer
지금 게시물은 busboy에 대한 이야기여서 사용법에 대한 설명은 하지 않는다.

### busboy와의 차이점
multer는 사실 busboy 위에서 동작하는 Express 미들웨어다.

```
HTTP Request
     ↓
   busboy
     ↓
   multer
```

즉 multer는 busboy를 기반으로 만든 추상화된 wrapper라고 볼 수 있다.
<br>

### 특징
장점
- Express 미들웨어 형태로 바로 사용 가능
- 파일 저장 로직이 단순
- 설정이 쉽다(이건 회사에서 쓴 경험이 있으니 알거다. 진짜진짜 쉽다.)

단점
- 스트림 직접 제어 어려움
- 파이프라인 커스터마이징 제한
- 복잡한 업로드 로직 구현이 어려움
<br><br>



## 사용 기준
일반적으로 다음 기준으로 선택한다.

### multer
- 일반적인 파일 업로드
- 간단한 업로드 API
<br>

### busboy
- 업로드 후 바로 S3 전송
- 업로드 → transform → 저장
- 대용량 파일 처리
- streaming pipeline 구성
- 업로드 흐름 직접 제어
<br><br>



## busboy 예제 코드
```ts
import busboy from "busboy";
import { createWriteStream } from "fs";
import path from "path";

async function uploadFile(req: Request) {
  const bb = busboy({
    headers: req.headers,
    // fileHwm: number, // 청크 크기 (highWaterMark는 스트림 전체 적용이다.)
    // limits: {
    //  fileSize: number, // 업로드 가능 최대 크기
    //  files: number, // 업로드 최대 개수
    // },
  });

  bb.on("file", (name, file, info) => {
    const { filename } = info;
    const safeFilename = path.basename(filename);

    const saveTo = `./uploads/${safeFilename}`;
    const writeStream = createWriteStream(saveTo);

    file.pipe(writeStream);

    file.on("end", () => {
      console.log(`Upload completed: ${safeFilename}`);
    });

    writeStream.on("error", (err) => {
      console.error("Write error:", err);
    });
  });

  bb.on("finish", () => {
    console.log("All uploads finished");
  });

  bb.on("error", (err) => {
    console.error("Busboy error:", err);
  });

  req.pipe(bb);
}
```
